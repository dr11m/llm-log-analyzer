## Задача
Объедини все анализы в один итоговый отчёт:

1. **Дедупликация**: одна и та же проблема может встречаться в нескольких чанках — объедини в одну, указав общее количество и полный временной диапазон.
2. **Тренды**: определи, ухудшается или улучшается ситуация со временем (сравни старые чанки с новыми).
3. **Общий статус системы**:
   - HEALTHY: нет серьёзных проблем, система работает в норме
   - DEGRADED: есть проблемы среднего уровня, система работает, но хуже нормы
   - CRITICAL: есть критичные проблемы, требуется немедленное вмешательство
4. **Нужна ли остановка (should_shutdown)**: true только если продолжение работы бессмысленно или вредно.
5. **Рекомендации**: конкретные действия, отсортированные по приоритету.
6. **Поиск аномалий между периодами**:
   - Резкие изменения в частоте ошибок
   - Появление новых типов ошибок
   - Изменение паттернов (время ответа, объёмы данных, активность)
   - Корреляции (проблема X начинается одновременно с проблемой Y)
   - Исчезновение событий, которые раньше были регулярными

## Формат ответа (строго JSON, без markdown-обертки)

## Правила
- issues отсортированы: CRITICAL первые, затем MEDIUM, затем LOW.
- Убери шум: единичные warnings без паттерна не включай.
- Каждая проблема должна иметь конкретные числа, не "множество".
- should_shutdown = true — крайняя мера, только если система действительно не может работать.
- anomalies — это межчанковые паттерны: если аномалий нет — верни пустой список.
