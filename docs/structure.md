# Структура проекта

## Назначение

AI Log Analyzer — модуль для анализа логов с помощью LLM. Проект состоит из двух основных потоков: генерация контекста по исходникам и регулярный анализ логов с формированием отчета.

## Основные директории и файлы

- `analyzer.py` — основной вход для анализа логов (manual/auto режимы).
- `generate_context.py` — генерация контекста проекта на базе исходников и документов.
- `src/` — основная логика приложения (оркестратор, чанкер, отчеты, LLM‑клиент).
- `prompts/` — шаблоны промптов для LLM по языкам (`ru/`, `en/`).
- `single_prompts/` — разовые промпты для ручного анализа в агентных средах.
- `data/` — генерируемые контекстные файлы и состояние анализа.
- `reports/` — выходные отчеты анализа логов.
- `config.example.yaml` — шаблон конфигурации.
- `config.yaml` — локальная конфигурация (не для публикации).
- `.env.example` — шаблон переменных окружения.
- `.env` — локальные секреты (не для публикации).
- `requirements.txt` — зависимости Python.
- `README.md`, `TODO.md`, `LICENSE` — документация и лицензия.
- `article/` — дополнительные материалы автора (не участвуют в выполнении кода).

## Основные модули (src)

| Модуль | Назначение |
|---|---|
| `src/orchestrator.py` | Координатор анализа: чанки → LLM → объединение → отчет → чекпоинт |
| `src/chunker.py` | Чтение логов с конца, чанкинг по строкам или токенам, поддержка ротации |
| `src/prompt_builder.py` | Сбор системного контекста и загрузка промптов |
| `src/report_generator.py` | Генерация JSON и Markdown отчетов |
| `src/state_manager.py` | Хранение состояния анализа (`state.json`), атомарная запись |
| `src/llm/base.py` | Абстрактный интерфейс LLM‑клиента |
| `src/llm/glm_client.py` | Реализация клиента для GLM‑4.7, ретраи, учет стоимости |
| `src/utils/logger.py` | Базовый логгер проекта |

## Данные и состояние

- `data/project_context.md` — контекст проекта для LLM (генерируется/редактируется).
- `data/analysis_rules.md` — правила анализа (генерируются/редактируются).
- `data/state.json` — чекпоинт: где остановился прошлый анализ.
- `reports/*.json` и `reports/*.md` — результаты анализа.

## Структура базы данных

В проекте **нет** базы данных. Все состояние и результаты хранятся в файловой системе.

## Внешние зависимости

- `requests` — HTTP‑клиент для LLM API.
- `pyyaml` — загрузка конфигурации.
- `python-dotenv` — загрузка переменных окружения.
